QVD ofrece una amplia gama de características que podrías esperar de cualquier otro VDI comercial.

Solucion VDI de alta densidad
~~~~~~~~~~~~~~~~~~~~~~~~~
QVD es una plataforma de virtualización de contenedores que es capaz de ejecutar múltiples máquinas virtuales a partir de una o más imágenes de sistema en un sólo nodo de la red. De hecho, QVD puede utilizar virtualización de contenedor ligero o puede ofrecer virtualización a gran escala, en función de sus necesidades. Al reutilizar una imagen de sistema base cargada en varias máquinas virtuales, puedes estandarizar el entorno que le proporcionas a tus trabajadores, facilitando la gestión y reduciendo significativamente los costos. QVD está diseñado para ejecutar múltiples escritorios por nodo de servidor para dar servicio a muchos usuarios.

Balanceo de carga y alta disponibilidad
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
En un entorno empresarial donde se espera que QVD soporte cientos de usuarios, es inviable ejecutar cada escritorio en el mismo nodo de servidor. QVD está diseñado para trabajar en clusters de nodos de servidor, cada uno ejecutando las máquinas virtuales para dar servicio a los distintos usuarios. QVD hace uso de su propio demonio de enrutamiento integrado de capa 7 para establecer conexiones y asegurarse de que un cliente que se conecta a cualquier nodo del servidor se redirige al nodo correcto a fin de acceder al escritorio apropiado. Si una máquina virtual para un usuario particular no se está ejecutando en un nodo de servidor disponible, QVD inteligente determina qué nodo de servidor tiene más recursos disponibles para iniciar la máquina virtual. Esto ayuda a asegurar la alta disponibilidad de todos los entornos de escritorio, incluso en el caso de que un nodo de servidor falle.

Auto Aprovisionamiento
~~~~~~~~~~~~~~~~~~~~~~
Simplemente estableciendo un nombre de usuario y contraseña en el entorno de QVD, y asignar al usuario la imagen de escritorio que se quiera, es posible la prestación de un escritorio en cuestión de segundos. QVD también se integra con sistemas de autenticación externos y admite la autenticación con LDAP. Esto significa que tan pronto como hayas agregado a un usuario a la plataforma QVD, puedes hacer uso del mecanismo de autenticación interna, o de integración con un proveedor externo, como Active Directory.
Además, las herramientas de línea de comandos y la capacidad para hacer scripts integrada en el producto, hace que sea muy fácil hacer un fichero por lotes para auto aprovisionar por ejemplo que cuando un usuario es añadido al entorno, se cree automáticamente un escritorio.

Soporte de Plataformas de Cliente
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
QVD actualmente ofrece el software de cliente para entornos Linux y Windows.. También hay un cliente en desarrollo de Android. QVD está diseñado para ser accesible a los usuarios desde cualquier sistema operativo base.
Permitiendo a los usuarios la flexibilidad para acceder a sus ordenadores desde cualquier sistema que tengan a mano. Simplemente ejecutando el cliente, el usuario puede tener acceso a un escritorio en cuestión de segundos. Debido al modelo cliente-servidor detrás QVD, es posible ver el potencial de recuperarte de desastres instantáneamente. Así que si el sistema de un usuario presenta una fallo de hardware grave, cualquier otro sistema se puede poner en marcha para que el usuario tenga acceso a todos sus datos y aplicaciones de forma instantánea.

Herramientas Web de Administración
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Mientras QVD proporciona herramientas de línea de comandos de administración para permitir flexibilidad a base de scripts, y para permitir a los administradores a trabajar a través de SSH y otras opciones de acceso remoto; QVD también proporciona un completo entorno web de administración proporcionando una interfaz gráfica de usuario completa para administrar y monitorizar toda la infraestructura.

La herramienta de administración Web de QVD hace que sea simple la gestión y la monitorización de tu infraestructura de QVD. Esta interfaz Web proporciona una ayuda a tus administradores de sistemas a mantener el control sobre los usuarios, las máquinas virtuales y las imágenes del sistema de manera fácil. Las Herramientas Web Administrativas incluso vienen con un applet de Java que se conecta directamente a la consola por puerto serie de cualquier máquina virtual en ejecución, de modo que un administrador puede conectarse rápidamente al entorno de un usuario y darle soporte.

Seguridad
~~~~~~~~~
QVD proporciona un alto nivel de seguridad dentro del entorno empresarial. Mediante la ejecución de la imagen del sistema en modo de sólo lectura en una máquina virtual, cualquier tipo de compromiso a nivel de sistema en una máquina virtual sólo afectará al escritorio mientras que la máquina virtual está en ejecución. Esto significa que si un escritorio se ve comprometida, simplemente reiniciar la máquina virtual es suficiente para devolverla a su estado original. Enjaulando a los usuarios dentro de sus propias máquinas virtuales, el comportamiento a nivel de usuario no puede afectar a otros usuarios, garantizando el buen funcionamiento de todos los escritorios. Por último, las conexiones de los clientes son encriptadas usando tecnología SSL/TLS para que todas las comunicaciones entre una estación de trabajo y el entorno de escritorio final vayan adecuadamente securizadas.

Acceso Remoto
~~~~~~~~~~~~~
QVD hace uso de una versión modificada del protocolo NX, lo que hace que se puede reducir masivamente la carga del ancho de banda al servir entornos de ventanas a través de Internet. QVD ofrece diferentes algoritmos de compresión en función de la configuración del usuario del ancho de banda disponible, lo que permite a los usuarios conectarse desde entornos con muy poco ancho de banda.
Con la configuración de red adecuada, los usuarios pueden acceder a sus escritorios de forma remota y segura, ya estén en su casa o en la carretera.
En lugar de configurar una VPN y servir ficheros a través de Internet, un usuario haciendo uso de QVD podrá acceder a su escritorio entero como si estuviera trabajando en local. Eso significa tener acceso a todos sus programas, carpetas compartidas en la red interna y sus datos personales.
